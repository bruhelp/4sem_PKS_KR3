# Симулятор лифта

## Описание проекта

Приложение написано для дисциплины Программирование корпоративных систем.
**Симулятор лифта** — консольный симулятор работы лифтов в многоэтажном здании, реализованный на языке **C++17** с использованием **многопоточности**.

Приложение моделирует многопоточную работу нескольких лифтов, обработку вызовов пассажиров, распределение заявок между лифтами различными алгоритмами и сбор статистики.


## Реализованные алгоритмы диспетчеризации

### FIFO

Обслуживание заявок в порядке их поступления.

### Nearest Car

Выбирается лифт, расположенный ближе всего к этажу вызова.

### Round Robin

Вызовы распределяются между лифтами по кругу.

### SCAN

Лифт продолжает движение в текущем направлении и обслуживает вызовы по пути следования.

## Как запустить?

### 1. Клонировать репозиторий

```bash
git clone <https://github.com/bruhelp/4sem_PKS_KR3.git>
```

### 2. Сборка


```bash
cmake -S . -B build -G "MinGW Makefiles"

или

cmake -S . -B build -G
```

```bash
cmake --build build
```

## Запуск

```bash
build/ElevatorSimulator.exe
```

![демонстрация](./work.gif)

После запуска программа предложит ввести параметры моделирования
в таком порядке:

* Количество этажей в здании (2-50)
* Количество лифтов в здании (1-6)
* Максимальная вместимость одного лифта (1-20)
* Пассажиров на одном этаже (1-50) — невлияющий параметр на данном этапе
* Скорость движения лифта за этаж (мс)
* Время выполнения симуляции (с)
* Начальное значение генератора случайных чисел, который необходим для сравнения разных алгоритмов

Пример:

```text
Floors (2-50): 13
Elevators (1-6): 4
Capacity (1-20): 12
Passengers per floor (1-50): 5
Travel time per floor (ms): 1500
Simulation duration (sec): 30
Random seed: 1
```

Затем необходимо будет выбрать один из четырёх алгоритмов на выбор.

## Отображаемая информация

Для каждого лифта выводятся:

* Номер лифта
* Текущий этаж
* Количество пассажиров
* Вместимость
* Текущее состояние
* Целевой этаж
* Маршрут обслуживания

Пример:

```text
Elevator #1 | Floor 4 | Load 3/10 | Target 1 | State Moving
Route: 1 -> 3 -> 1
```

## Статистика симуляции

После завершения моделирования отображаются:

* Используемый алгоритм
* Количество обслуженных пассажиров
* Среднее время ожидания
* Максимальное время ожидания
* Среднее время поездки
* Общее время движения лифтов
* Общее время обслуживания пассажиров
* Коэффициент загрузки системы
* Время выполнения симуляции

Пример:

```text
Simulation finished

Algorithm: FIFO
Passengers served: 120
Average wait time: 2.3 sec
Maximum wait time: 8 sec
Average ride time: 5.7 sec
Movement time: 95000 ms
Service time: 30000 ms
Utilization: 79.2%
Execution time: 31.4 sec
```

## Файлы результатов

После запуска автоматически создаётся каталог:

```text
results/
```

В нём сохраняются:

### Журнал событий

```text
results/events.log
```

Содержит:

* создание вызовов;
* назначение лифтов;
* движение между этажами;
* открытие и закрытие дверей;
* завершение симуляции.

### История запусков

```text
results/simulation_history.txt
```

Содержит:

* параметры запуска;
* выбранный алгоритм;
* статистику работы системы.
