# Симулятор лифта

## Описание проекта

Приложение написано для дисциплины Программирование корпоративных систем.
**Симулятор лифта** — консольный симулятор работы лифтов в многоэтажном здании, реализованный на языке **C++17** с использованием **многопоточности**.

Приложение моделирует многопоточную работу нескольких лифтов, обработку вызовов пассажиров, распределение заявок между лифтами различными алгоритмами и сбор статистики.


## Реализованные алгоритмы диспетчеризации

### FIFO

Обслуживание заявок в порядке их поступления.

### Nearest Car

Выбирается лифт, расположенный ближе всего к этажу вызова.

### Round Robin

Вызовы распределяются между лифтами по кругу.

### SCAN

Лифт продолжает движение в текущем направлении и обслуживает вызовы по пути следования.

## Запуск

```bash
build/ElevatorSimulator.exe
```

![демонстрация](./work.gif)

После запуска программа предложит ввести параметры моделирования
в таком порядке:

* Количество этажей в здании (2-50)
* Количество лифтов в здании (1-6)
* Максимальная вместимость одного лифта (1-20)
* Пассажиров на одном этаже (1-50) — невлияющий параметр на данном этапе
* Скорость движения лифта за этаж (мс)
* Время выполнения симуляции (с)
* Начальное значение генератора случайных чисел, который необходим для сравнения разных алгоритмов

Пример:

```text
Floors (2-50): 13
Elevators (1-6): 4
Capacity (1-20): 12
Passengers per floor (1-50): 5
Travel time per floor (ms): 1500
Simulation duration (sec): 30
Random seed: 1
```

Затем необходимо будет выбрать один из четырёх алгоритмов на выбор.

## Интеграционные сценарии


### Сценарий 1: Одиночный вызов

Цель: проверить базовую логику: вызов, посадка, перемещение, высадка одного пассажира.

Этапы:
* Создаётся один лифт вместимостью 5 человек.
* Генерируется один пассажир на 1-м этаже, который хочет попасть на 5-й этаж.
* Вызов обрабатывается диспетчером (стратегия FIFO).
* Лифт приезжает на 1-й этаж, пассажир заходит.
* Лифт перемещается на 5-й этаж, пассажир выходит.
* Проверяется, что пассажир доставлен (список завершённых пассажиров не пуст).

Ожидаемый результат:
```
Вывод SUCCESS: Passenger delivered.
```


### Сценарий 2: Переполнение лифта

Цель: проверить, что лифт не берёт больше пассажиров, чем позволяет вместимость, и оставшиеся дожидаются следующего рейса.

Этапы:
* Создаётся лифт вместимостью 2 человека.
* На 1-м этаже генерируется 3 пассажира, все хотят ехать на 3-й этаж.
* Диспетчер назначает вызовы этому лифту.
* Лифт прибывает на 1-й этаж, забирает только 2 пассажиров (третий остаётся ждать).
* Лифт отвозит первых двух на 3-й этаж, высаживает.
* Лифт возвращается на 1-й этаж, забирает третьего пассажира и отвозит его.
* Проверяется, что все 3 пассажира доставлены.

Ожидаемый результат:

```
Вывод SUCCESS: All passengers delivered.
```

### Сценарий 3: Ближайший лифт (Nearest Car)

Цель: проверить, что стратегия NearestCarStrategy выбирает лифт, который находится ближе всего к этажу вызова.

Этапы:
* Создаются два лифта:
* Лифт №1 находится на 1-м этаже.
* Лифт №2 перемещается на 10-й этаж (за счёт добавления назначения и вызова moveOneFloor()).
* Поступает вызов с 5-го этажа.
* Стратегия NearestCarStrategy вычисляет расстояния:
* Для лифта №1: |5 – 1| = 4
* Для лифта №2: |5 – 10| = 5
* Выбирается лифт №1.
* Проверяется, что назначенный лифт имеет ID = 1.

Ожидаемый результат:

```
Вывод SUCCESS: Nearest elevator selected.
```

### Сценарий 4: SCAN и встречные вызовы

Цель: проверить, что стратегия SCAN учитывает направление движения: если лифт движется вверх, он не будет назначен на вызов, поступающий с этажа, расположенного выше, если пассажиру нужно ехать вниз (несовместимое направление).

Этапы:
* Создаются два лифта:
* Лифт №1 поднимается на 10-й этаж и продолжает движение вверх (есть назначение на 12-й этаж).
*  Лифт №2 находится в свободном состоянии (Idle) на 1-м этаже.
* Поступает вызов с 5-го этажа вниз (на 1-й этаж).
* SCAN проверяет совместимость:
* Лифт №1 (направление Up) — этаж вызова (5) ниже текущего этажа (10) → несовместим.
* Лифт №2 (Idle) — совместим.
* Выбирается лифт №2.
* Проверяется, что назначенный лифт имеет ID = 2.

Ожидаемый результат:

```
Вывод SUCCESS: SCAN picked idle elevator over incompatible moving one.
```

### Сценарий 5: Конфигурация с граничными значениями

Цель: убедиться, что симулятор корректно работает с минимальными и максимальными допустимыми параметрами конфигурации.

Этапы:
* Создаётся конфигурация с минимальными значениями:
* Этажи = 2, лифты = 1, вместимость = 1, пассажиров на этаж = 1, длительность = 1 секунда.
* Запускается симуляция с одним пассажиром.
* Создаётся конфигурация с максимальными значениями:
* Этажи = 50, лифты = 6, вместимость = 20, пассажиров на этаж = 50.
* Снова запускается симуляция.
* В обоих случаях проверяется, что пассажир доставлен без ошибок.

Ожидаемый результат:

```
Вывод SUCCESS: Boundary configs work.
```
